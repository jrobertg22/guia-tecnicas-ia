Tema 06: Deep Learning y Reinforcement Learning

6.2. El papel del deep learning dentro del machine learning

El aprendizaje automático (Machine Learning) se subdivide en varias categorías principales:

  • Clásico: Supervisado (datos etiquetados), No supervisado (descubrir estructuras/agrupaciones) y Semisupervisado.
  • Ensemble Learning: Combinación de modelos para mejorar precisión.
  • Deep Learning (Aprendizaje Profundo): Útil frente a datos muy complejos o cuando se buscan precisiones superiores al ML clásico.
  • Reinforcement Learning (Aprendizaje por Refuerzo): Utilizado cuando no hay datos de entrada previos, sino un entorno interactivo.

6.3. Redes neuronales y deep learning

Definición: El Deep Learning es una clase de algoritmos que utiliza múltiples capas para extraer progresivamente características de nivel superior de la entrada bruta (ej. en una imagen: detectar bordes en las primeras capas y objetos completos en las últimas).
  • Resurgimiento: Tienen éxito hoy en día gracias a la enorme capacidad de computación (CPU, GPU de alto paralelismo y TPU especializadas) y al almacenamiento masivo de datos (Big Data).
  • Versatilidad: Pueden actuar como sustitutos de métodos supervisados, no supervisados y semisupervisados, resolviendo problemas de PNL, visión por computador, reconocimiento de voz, etc.

6.4. Redes prealimentadas profundas (Feedforward)

En estas redes, la información fluye en una sola dirección: de la entrada a la salida, sin bucles ni retroalimentación.

  • El Perceptrón: Unidad básica (monocapa) que funciona como un clasificador lineal separando clases mediante un hiperplano, utilizando funciones de activación (signo, escalón, logística).
  • Perceptrón Multicapa (MLP): Red profunda con al menos una capa oculta. Se entrena iterativamente ajustando pesos mediante el método de retropropagación del error (propagando el gradiente de error hacia atrás).

6.5. Redes neuronales recurrentes profundas (RNN)

A diferencia de las feedforward, las RNN tienen neuronas específicas con una "memoria" que almacenan entradas de ciclos anteriores, permitiendo bucles de retroalimentación.

  • Aplicación ideal: Procesamiento de secuencias de datos, como el texto, la genómica, la traducción o la síntesis de voz.
  • LSTM (Long Short-Term Memory): Mejoran a las RNN incorporando "puertas" (entrada, salida y olvido) que controlan qué información guardar o descartar.
  • GRU (Gated Recurrent Units): Variación de LSTM con solo dos puertas, mejorando el rendimiento computacional, aunque requiriendo redes más grandes para igualar su expresividad.

6.6. Autoencoders (AE)

Redes neuronales simétricas en forma de "reloj de arena", utilizadas típicamente en tareas no supervisadas.

  • Estructura: Comprimen la información desde la entrada (codificador) hacia un código central o "cuello de botella", para luego intentar reconstruirla a la salida (decodificador).
  • Variantes: DAE (Denoising Autoencoders) para eliminar ruido; SAE (Sparse) para extraer características; y VAE (Variational) basados en probabilidad y matemáticas bayesianas para generar nuevos datos.

6.7. Redes neuronales convolucionales (CNN)

CNN / DCNN: Son el estándar de oro en visión artificial (búsqueda de objetos, reconocimiento facial). En lugar de procesar la imagen completa de golpe, aplican filtros ("convoluciones") por bloques pequeños de píxeles, extrayendo características abstractas capa por capa.
  • Redes Deconvolucionales (DNN): Operan a la inversa. Al alimentarlas con una etiqueta ("gato"), generan una imagen sintética del objeto.

6.8. Redes generativas antagónicas (GAN)

Consisten en dos redes neuronales compitiendo entre sí en un entorno de "juego de suma cero".

  • Red Generativa: Intenta crear datos o imágenes falsas lo más realistas posible.
  • Red Discriminativa: Intenta detectar si la imagen que recibe es real (del dataset) o falsa (creada por el generador).
  • Aplicaciones: Deepfakes (sustitución de caras en vídeo), envejecimiento facial, transferencia de estilo y generación de imágenes artificiales ultrarrealistas.

6.9. Aprendizaje por refuerzo (Reinforcement Learning)

El agente aprende a tomar decisiones interactuando con un entorno a base de prueba y error, buscando maximizar una recompensa (ej. videojuegos o coches autónomos).

  • Compromiso Exploración/Explotación: El agente debe decidir entre aprovechar el conocimiento seguro que ya le da puntos (explotar) o arriesgarse a tomar rutas nuevas para descubrir recompensas mayores (explorar). Se maneja comúnmente con la estrategia epsilon-greedy.
  • Procesos de Decisión de Markov (MDP): Modelo matemático donde se consideran estados, transiciones y un factor de descuento para valorar las recompensas futuras.
  • Q-Learning: Algoritmo clásico (libre de modelo) que guarda los valores aprendidos en una matriz gigante (Q-matrix) para saber qué acción es la óptima en cada estado.

6.10. Aprendizaje por refuerzo profundo (Deep RL)

Combina el Reinforcement Learning con Redes Neuronales Profundas. Permite operar en entornos donde el número de estados posibles es inmanejable para una tabla (como el juego de Go o los píxeles de un videojuego de Atari).

  • Deep Q-Networks (DQN): Sustituyen la tabla Q del Q-learning tradicional por una red neuronal que estima el valor Q.
  • Double DQN (DDQN): Soluciona el problema de "sobreestimación" de recompensas de DQN usando dos redes neuronales separadas: una para seleccionar la acción y otra para evaluar su calidad.
  • Casos de éxito: AlphaGo (y sus versiones posteriores AlphaGo Zero sin datos humanos) de DeepMind, que logró vencer a campeones mundiales de Go.

6.11. Ejemplos de implementación

Para construir modelos de Deep Learning en código, la industria emplea frameworks especializados como TensorFlow (creado por Google) y su API de alto nivel Keras.

  • Tensor (TensorFlow): Es una entidad algebraica generalizada (escalar, vector o matriz multidimensional) independiente de cualquier sistema de coordenadas.
  • Flujo típico (Ejemplo Fashion-MNIST):
    1. Carga y preprocesamiento de datos (normalización de píxeles a valores entre 0 y 1).
    2. Definición del modelo Secuencial: Capa de aplanado (Flatten) seguida de capas densas (Dense con activación ReLU).
    3. La última capa utiliza la función Softmax, la cual devuelve un array de probabilidades normalizadas para clasificar la entrada.
    4. Compilación (definiendo la función de pérdida y optimizador como Adam) y Entrenamiento (model.fit) validando la exactitud y el posible sobreajuste (overfitting).
